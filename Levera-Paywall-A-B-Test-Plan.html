# Levera — Paywall A/B Test Plan *Версия 1.0 — Протокол тестирования цены и момента paywall* --- ## Что тестируем Два независимых вопроса. Каждый — отдельный тест. | Тест | Вопрос | Переменная | |---|---|---| | **Тест 1** | Какая цена конвертирует лучше? | ₽349/мес vs ₽499/мес | | **Тест 2** | Какой момент показа paywall конвертирует лучше? | После W1 Weekly Report vs После первого барабана | Запускать тесты **последовательно**, не параллельно. Иначе результаты смешаются. --- ## Тест 1 — Цена ### Гипотеза Monetisation Strategy предполагает: если конверсия при ₽499 падает менее чем на 20% — переходить на ₽499. Это +43% к MRR при той же базе. **H0 (нулевая):** Конверсия free → paid одинакова при ₽349 и ₽499. **H1 (альтернативная):** Конверсия при ₽499 падает менее чем на 20% → ₽499 выгоднее по MRR. ### Дизайн теста | Параметр | Вариант A | Вариант B | |---|---|---| | Цена месячного плана | ₽349/мес | ₽499/мес | | Цена годового плана | ₽2 990/год | ₽3 990/год | | Всё остальное | Идентично | Идентично | **Разделение:** 50% новых пользователей → A, 50% → B. Разделение по user_id (чётный/нечётный). **Момент показа paywall:** фиксированный для обоих вариантов — после первого Weekly Report (конец W1). ### Размер выборки Минимальный размер выборки для статистической значимости (95% уверенность, 20% MDE): При ожидаемой базовой конверсии ~8%: - Минимум: **385 пользователей на каждый вариант** = 770 всего - При воронке: ~30% установок доходят до paywall → нужно ~2 570 установок **Реалистичный срок:** при 200–300 установках/мес → тест займёт 10–13 недель. **Ускорить:** через Telegram-блогера (один пост = 500–2000 установок). ### Метрики | Метрика | Приоритет | Определение | |---|---|---| | **Conversion rate** | Primary | % пользователей, перешедших в платный тир | | **Revenue per install** | Primary | Средний доход с одной установки | | **Time to convert** | Secondary | Медианное время от установки до оплаты | | **Annual plan %** | Secondary | % из платящих, выбравших годовой план | | **W4 retention (платные)** | Secondary | Остаются ли платящие пользователи дольше | ### Критерий победы ``` Если conv(B) >= conv(A) × 0.80: → Вариант B побеждает. Перейти на ₽499. → Новый MRR = MRR(A) × (conv(B)/conv(A)) × (499/349) Если conv(B) < conv(A) × 0.80: → Вариант A побеждает. Остаться на ₽349. → Переосмыслить ценовую стратегию через 6 месяцев при росте базы. ``` **Пример расчёта:** - A: 100 пользователей, 8% конверсия = 8 платящих × ₽349 = ₽2 792 - B: 100 пользователей, 7% конверсия (−12.5%) = 7 платящих × ₽499 = ₽3 493 (+25%) - **B выигрывает** несмотря на меньшую конверсию. --- ## Тест 2 — Момент показа Paywall ### Гипотеза Monetisation Strategy описывает 4 возможных момента. Два наиболее вероятных кандидата: - **Момент A (After Weekly Report):** родитель видит прогресс ребёнка → понимает ценность → готов платить за продолжение - **Момент B (After First Spin):** ребёнок получил первый приз из барабана → эмоциональный пик → родитель не хочет останавливать **H0:** Момент показа не влияет на конверсию. **H1:** Один момент конвертирует значимо лучше другого. ### Дизайн теста | Параметр | Вариант A | Вариант B | |---|---|---| | Момент показа | После первого Weekly Report (конец W1) | После первого spin барабана | | Цена | Фиксированная (победитель из Теста 1) | Фиксированная (победитель из Теста 1) | | Триггер | Открытие P-13 (Weekly Report) | Закрытие C-06 (Барабан) с первым призом | **Запускать Тест 2 только после завершения Теста 1.** ### Метрики Те же, что в Тесте 1. Дополнительно: | Метрика | Зачем | |---|---| | **Paywall skip rate** | % пользователей, закрывших paywall без действия | | **Paywall → trial rate** | % открывших бесплатный пробный период (если добавить) | | **Days to convert** | Быстрее ли конвертируют после барабана, чем после отчёта? | --- ## Дополнительные тесты (после основных двух) ### Тест 3 — Free trial vs нет | Вариант A | Вариант B | |---|---| | Paywall без trial | 7-дневный бесплатный trial Family плана | | Карта нужна | Карта не нужна | Гипотеза: trial без карты увеличивает конверсию за счёт снижения барьера. Риск: выше % пустых trial без конверсии. **Метрика победы:** Trial conversion rate ≥ 35% (стандарт для education apps с trial). ### Тест 4 — Формат предложения на экране paywall | Вариант A | Вариант B | |---|---| | Месячный план выделен первым | Годовой план выделен первым | | Месяц: ₽349, год: ₽2 990 (−28%) | Год: ₽2 990, месяц: ₽349 (ещё +₽1 200/год) | Гипотеза: выделение годового плана первым увеличивает долю годовых подписок → снижает churn. --- ## Инфраструктура тестирования ### Техническая реализация ```javascript // При регистрации пользователя — назначить группу: const testGroup = userId % 2 === 0 ? 'A' : 'B' await supabase .from('users') .update({ ab_test_group: testGroup }) .eq('id', userId) // При рендеринге экрана paywall: const price = user.ab_test_group === 'A' ? 349 : 499 const annualPrice = user.ab_test_group === 'A' ? 2990 : 3990 ``` ### Что логировать ```javascript // Событие: paywall показан logEvent('paywall_shown', { user_id, ab_group, trigger: 'weekly_report' | 'first_spin', timestamp }) // Событие: пользователь тапнул на план logEvent('paywall_tap', { user_id, ab_group, plan: 'monthly' | 'annual', price }) // Событие: оплата прошла logEvent('subscription_started', { user_id, ab_group, plan, price, revenue }) ``` ### Инструменты | Инструмент | Для чего | Стоимость | |---|---|---| | Supabase | Хранение ab_test_group, логи событий | Бесплатно | | RevenueCat | Биллинг + аналитика подписок | Бесплатно до $2 500 MRR | | PostHog (self-hosted) | Event tracking, A/B анализ | Бесплатно | | Google Sheets | Dashboard для ручного анализа | Бесплатно | --- ## Расписание тестов | Период | Действие | |---|---| | До запуска монетизации | Настроить ab_test_group в БД.
