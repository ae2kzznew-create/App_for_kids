бесплатный tier | --- ## Схема базы данных (Supabase) ### Таблица: `users` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() email text UNIQUE phone text UNIQUE role text CHECK (role IN ('parent', 'child')) created_at timestamptz DEFAULT now() ``` ### Таблица: `families` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() name text created_at timestamptz DEFAULT now() ``` ### Таблица: `family_members` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() family_id uuid REFERENCES families(id) user_id uuid REFERENCES users(id) role text CHECK (role IN ('parent', 'child')) display_name text age int -- только для детей, нужен для ZPD-калибровки avatar_url text created_at timestamptz DEFAULT now() ``` ### Таблица: `quests` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() family_id uuid REFERENCES families(id) child_id uuid REFERENCES family_members(id) title text description text skill_category text CHECK (skill_category IN ('household', 'academic', 'emotional', 'physical', 'social')) zpd_level int CHECK (zpd_level IN (0, 1, 2, 3)) -- 3=новичок, 0=мастер xp_reward int DEFAULT 10 status text CHECK (status IN ('active', 'pending_confirm', 'completed', 'skipped')) created_at timestamptz DEFAULT now() assigned_date date ``` ### Таблица: `quest_completions` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() quest_id uuid REFERENCES quests(id) child_id uuid REFERENCES family_members(id) completed_at timestamptz DEFAULT now() confirmed_by uuid REFERENCES family_members(id) -- parent_id confirmed_at timestamptz photo_url text -- фото триумф-момента (опционально) triumph_word text -- слово/эмоция после квеста (опционально) xp_granted int auto_confirmed boolean DEFAULT false -- true если авто-подтверждение через 24ч ``` ### Таблица: `child_progress` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() child_id uuid REFERENCES family_members(id) current_xp int DEFAULT 0 current_level int DEFAULT 1 keys_available int DEFAULT 0 -- ключики для барабана streak_days int DEFAULT 0 last_active date skill_scores jsonb -- {"household": 75, "academic": 40, ...} 0-100 ``` ### Таблица: `wishes` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() child_id uuid REFERENCES family_members(id) title text rarity text CHECK (rarity IN ('common', 'rare', 'epic')) value_type text CHECK (value_type IN ('money', 'time', 'emotion')) is_active boolean DEFAULT true is_postponed boolean DEFAULT false postponed_until date postpone_narrative text -- "Собачки улетели на луну..." created_at timestamptz DEFAULT now() ``` ### Таблица: `wheel_spins` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() child_id uuid REFERENCES family_members(id) wish_id uuid REFERENCES wishes(id) -- выпавший приз spun_at timestamptz DEFAULT now() key_used boolean DEFAULT true status text CHECK (status IN ('pending', 'fulfilled', 'postponed')) fulfilled_at timestamptz fulfillment_photo_url text family_points_awarded int DEFAULT 0 ``` ### Таблица: `family_bank` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() family_id uuid REFERENCES families(id) total_points int DEFAULT 0 updated_at timestamptz DEFAULT now() ``` ### Таблица: `notifications_log` ```sql id uuid PRIMARY KEY DEFAULT gen_random_uuid() recipient_id uuid REFERENCES users(id) type text -- 'quest_complete', 'prize_won', 'reminder', 'weekly_report' payload jsonb sent_at timestamptz DEFAULT now() read_at timestamptz ``` --- ## Логика уровней и ключиков ``` XP_PER_QUEST = 10 (базовое значение, варьируется по сложности) XP_TO_LEVEL = [0, 50, 120, 210, 320, 450, ...] -- Формула: XP_TO_LEVEL[n] = XP_TO_LEVEL[n-1] + (40 + n * 10) При достижении нового уровня: child_progress.current_level += 1 child_progress.keys_available += 1 child_progress.current_xp = остаток после порога → push родителю: "Миша вырос до уровня N! Ему выпал ключик 🗝️" → push ребёнку: триумф-экран с ключиком ``` --- ## Логика барабана (Wish Wheel) ### Вероятности по умолчанию ```javascript const RARITY_WEIGHTS = { common: 70, // ~70% rare: 25, // ~25% epic: 5, // ~5% } function spinWheel(childId) { // 1. Получить активные желания (is_active = true, is_postponed = false) const activeWishes = await getActiveWishes(childId) // 2. Если нет активных — вернуть заглушку (не должно случаться) if (!activeWishes.length) return null // 3. Построить взвешенный пул const pool = [] for (const wish of activeWishes) { const weight = RARITY_WEIGHTS[wish.rarity] for (let i = 0; i < weight; i++) pool.push(wish) } // 4. Выбрать случайно const result = pool[Math.floor(Math.random() * pool.length)] // 5. Списать ключик await decrementKeys(childId) // 6. Записать спин await createWheelSpin(childId, result.id) // 7. Push родителю await sendPush(parentId, 'prize_won', { child: name, prize: result.title, rarity: result.rarity }) return result } ``` ### Гарантия супер-редкого приза ```javascript // При каждом спине проверять: const daysSinceLastEpic = await getDaysSinceLastEpic(childId) const epicGuaranteeDays = family.epic_guarantee_days // задаёт родитель, default: 180 if (daysSinceLastEpic >= epicGuaranteeDays) { // Форсировать epic из пула const epicWishes = activeWishes.filter(w => w.rarity === 'epic') if (epicWishes.length) { result = epicWishes[Math.floor(Math.random() * epicWishes.length)] } } ``` ### Отсрочка (postpone) ```javascript // Родитель ставит приз на паузу: async function postponeWish(wishId, daysCount, narrative) { await supabase .from('wishes') .update({ is_postponed: true, postponed_until: addDays(new Date(), daysCount), postpone_narrative: narrative }) .eq('id', wishId) } // В spinWheel фильтровать: is_postponed = false ИЛИ postponed_until < now() // При выпадении отложенного приза (если случайно попал) — показать нарратив, // перекрутить автоматически ``` --- ## Family streak ```javascript // При каждом confirmed quest_completion: async function updateStreak(childId) { const progress = await getChildProgress(childId) const today = new Date().toISOString().split('T')[0] if (progress.last_active === yesterday(today)) { // Стрик продолжается await incrementStreak(childId) } else if (progress.last_active === today) { // Уже засчитано сегодня, ничего не делать } else { // Стрик сбросился await resetStreak(childId) } await updateLastActive(childId, today) } ``` **Авто-подтверждение (fallback):** ```javascript // Cron job каждые 24 часа: // Найти все quest_completions со статусом 'pending_confirm' старше 24 часов // → confirmed_at = now(), auto_confirmed = true, XP зачислен // → updateStreak(childId) ``` --- ## ZPD-калибровка ```javascript // При создании квеста — уровень скаффолдинга определяется автоматически: function getZpdLevel(childAge, skillScore, skill) { // skillScore: 0–100, берётся из child_progress.skill_scores[skill] if (skillScore < 25) return 3 // новичок — полная инструкция if (skillScore < 50) return 2 // знакомо — чеклист if (skillScore < 75) return 1 // умеет — только цель return 0 // мастер — без инструкции } // После выполнения квеста — обновить skill_score: async function updateSkillScore(childId, skill, zpd_level) { const delta = { 3: 5, 2: 8, 1: 12, 0: 15 }[zpd_level] // Чем выше уровень мастерства (ниже zpd) — тем меньше прироста (навык стабилизируется) await incrementSkillScore(childId, skill, delta) } ``` --- ##
