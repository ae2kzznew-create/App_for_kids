# Levera — UX Flow & Screen Map *Составлено Draper — на основе Validated Biz Plan и Brand Identity Document* --- ## Обзор архитектуры Три уровня игровой петли. Один продукт. | Петля | Игрок | Суть | |---|---|---| | 🧒 Личная | Ребёнок | Квесты → опыт → уровень → ключик → личный барабан → приз | | 👨‍👩‍👧 Семейная | Семья вместе | Реализованный приз + фото → семейный банк → семейный барабан | | 🌍 Внешняя | Семья + партнёры | Семейные баллы → конкурсы, гранты, коллаборации | --- ## Петля 1 — Личная (ребёнок) ### Механика ``` Квесты → Опыт → Уровень → Ключик → Барабан → Приз → Реализация ``` 1. Ребёнок выполняет квесты → копит опыт (XP) 2. Достигает порога → повышает уровень → получает **ключик** 3. Ключик = право крутить личный барабан 4. Барабан крутится → выпадает приз из списка (составленного вместе с родителем при онбординге) 5. Родитель получает уведомление: «Миша поднял 1 уровень. Ему выпала шоколадка» 6. Родитель реализует приз 7. Ребёнок может напомнить через приложение: **«Эй! Где моя шоколадка?»** — родителю приходит мягкое уведомление на понятном языке: *«Миша ждёт. Выполни своё обещание.»* ### Три уровня редкости призов | Уровень | Символ | Шанс | Пример | |---|---|---|---| | Обычный | ⚪ | Часто (~70%) | Шоколадка, сок, наклейки | | Редкий | 🟡 | Редко (~25%) | Пиццерия, кино, новая книга | | Супер-редкий | 🔴 | Очень редко (~5%) | Дорогой набор, поездка, особое событие | **Гарантия:** супер-редкий приз выпадает минимум раз в [период, заданный родителем — например раз в полгода]. ### Три измерения ценности приза | Измерение | Суть | Пример | |---|---|---| | 💰 Деньги | Родитель тратит деньги | Шоколадка, LEGO, поездка | | ⏱️ Время | Родитель тратит время | Час вместе без телефона, совместный поход | | ❤️ Эмоции | Родитель отдаёт внимание | 10 обнимашек, особый ужин, семейный ритуал | Важно: самые редкие призы не обязаны быть дорогими. «Целый день с папой без дел» — супер-редкий приз с нулевым бюджетом. ### Отсрочка (вместо «нет») Родитель может временно убрать конкретный приз из пула. Приз не исчезает — он на паузе. **Нарратив для ребёнка:** *«На этой неделе собачки улетели на луну. У нас неделя космоса — шанс получить космические призы вырос!»* Вероятность других призов перераспределяется автоматически. Ребёнок слышит историю, не «нет». **Стоимость:** чем дольше пауза — тем дороже (escalating cost). **Привязка к дате:** родитель может зарезервировать выпадение конкретного приза к нужной дате — день рождения, Новый год. ### История роста (личный дневник) Каждое выполненное задание, каждый полученный приз, каждая реализованная награда — сохраняется в личной истории ребёнка. - Хронологическая лента: квесты, уровни, призы, фото - Визуализация роста навыков во времени - «Посмотри, каким ты был год назад» — момент гордости, а не ностальгии Это не только продуктовый механизм. Это артефакт, который семья хранит. --- ## Петля 2 — Семейная (совместная) ### Механика ``` Семейный приз реализован → Фото вместе → Семейный банк → Семейный барабан ``` 1. Когда семья реализует совместный приз (пиццерия, поход, кино) → делают **общую фотографию** в приложении 2. Фото подтверждает реализацию → в **семейный банк** начисляются баллы 3. При накоплении достаточного количества баллов → **семейный ритуал**: вся семья вместе крутит семейный барабан 4. Семейный барабан — отдельный пул призов: семейные активности, события, бонусы ### Семейный банк | Источник баллов | Сколько | |---|---| | Реализован обычный приз + фото | Мало | | Реализован редкий приз + фото | Средне | | Реализован супер-редкий приз + фото | Много | | Семейный квест выполнен вместе | Бонус | ### Кризисный механизм Если родитель временно не может реализовать приз (финансовые трудности, форс-мажор) — семейный банк может «спасти»: - Покрыть стоимость обычного приза баллами - Заменить денежный приз на эмоциональный или временной эквивалент - Дать родителю «кризисную карту» — отсрочку без escalating cost в исключительной ситуации Родитель активирует кризисный механизм сам. Никакой стигмы — просто инструмент. --- ## Петля 3 — Внешняя (партнёрская) *Долгосрочное направление. Не MVP.* Семейные баллы из банка открывают доступ к внешнему слою: - **Семейные конкурсы** — участие в конкурсах среди семей Levera - **Партнёрские призы** — бренды и сервисы предоставляют реальные призы для семейного барабана - **Гранты** — семьи с высокими показателями вовлечённости могут подавать на образовательные гранты - **Институт семьи** — коллаборации с государственными и НКО-партнёрами для поддержки семейных ценностей Это не монетизация. Это позиционирование Levera как инфраструктуры для семьи, а не просто приложения. --- ## User Flow — Родитель (обновлено) ``` [Onboarding] │ ▼ 1. Регистрация (телефон/email) │ ▼ 2. Добавление ребёнка (имя, возраст) │ ▼ 3. Wish Wheel Setup (совместный ритуал): — ребёнок диктует все желания (голос/текст) — родитель: ✅/❌, редкость (⚪🟡🔴), измерение (💰⏱️❤️) │ ▼ 4. AI-сессия: «Что хочешь развить в ребёнке?» (голос/текст) │ ▼ 5. AI предлагает квесты → родитель одобряет одним тапом │ ▼ [Ежедневный ритуал — Родитель] │ ▼ 6. Уведомление: «Миша выполнил задание» → тап-подтверждение │ ▼ 7. Dashboard: Skill Health Map + прогресс ключиков ребёнка │ ▼ [Когда ребёнок крутит барабан] │ ▼ 8. Уведомление: «Мише выпал приз: [название]» → родитель видит измерение (💰/⏱️/❤️) и что нужно сделать │ ▼ 9. Родитель реализует приз → отмечает в приложении (или получает напоминание от ребёнка: «Эй! Где мой приз?») │ ▼ [Семейный ритуал — при накоплении семейных баллов] │ ▼ 10. Уведомление: «Семейный барабан готов — крутите вместе!» │ ▼ 11. Семья крутит барабан → семейный приз → фото → баллы в банк │ ▼ [Еженедельно] │ ▼ 12. Weekly Report: достижение → прогресс → рекомендация AI ``` --- ## User Flow — Ребёнок (обновлено) ``` [Первый вход] │ ▼ 1. Приветственный экран: «Привет, [имя]. Ты уже назвал свои мечты. Зарабатывай ключики!» │ ▼ 2. Туториал (3–4 экрана): квест → опыт → уровень → ключик → барабан → приз │ ▼ [Ежедневный ритуал] │ ▼ 3. Главный экран: Quest-лист + прогресс опыта до следующего уровня │ ▼ 4. Квест → выполнил → «Готово» │ ▼ 5. Родитель подтверждает → XP зачислен → family streak +1 │ ▼ [Достиг нового уровня] │ ▼ 6. Анимация: «Уровень [N]! Тебе выпал ключик 🗝️» │ ▼ 7. Барабан разблокирован → крутит → приз выпал │ ▼ 8. Экран приза: что выпало, измерение, «Папа/мама уже знают» │ ▼ 9. Опционально: «Папа ещё не дал приз?» → кнопка напоминания │ ▼ [Семейный ритуал — совместно] │ ▼ 10. Семья вместе крутит семейный барабан → фото → баллы │ ▼ [Еженедельно] │ ▼ 11. Skill Tree + личная история роста ``` --- ## Screen-by-Screen: Родитель --- ### 🔐 P-01 — Регистрация **Что:** телефон/email, код подтверждения. Минимум полей. **Почему:** первый экран задаёт тон доверия. **Дальше:** → P-02 --- ### 👶 P-02 — Добавление ребёнка **Что:** имя, возраст (слайдер 6–17), фото опционально. **Почему:** возраст — единственный обязательный параметр для ZPD-калибровки. **Дальше:** → P-03 --- ### 🎡 P-03 — Wish Wheel Setup (онбординг-ритуал) **Что на экране:** «Пусть [имя] скажет всё, чего хочет. Без ограничений.» — Кнопка микрофона / текстовое поле — Карточки желаний появляются по мере ввода — Родитель проходит по каждой: ✅/❌ → редкость (⚪🟡🔴) → измерение (💰⏱️❤️) **Почему:** ребёнок видит, что родитель читает каждое желание и принимает решение честно. Это первый совместный ритуал в приложении. **Дальше:** → P-04 --- ### 🎙️ P-04 — AI-сессия **Что:** голос или текст. «Что хочешь развить в [имя]?» Примеры-триггеры: «Сам убирает», «Не умеет планировать», «Боится ответственности». AI → 1–2 уточняющих вопроса → 3–5 квестов. **Дальше:** → P-05 --- ### ✅ P-05 — Одобрение квестов **Что:** карточки квестов — название, описание, сложность, вклад в XP. «Принять все» / редактировать. **Дальше:** → P-06 --- ### 📊 P-06 — Dashboard **Что:** - Skill Health Map: 4–6 категорий → 🟢/🟡/🔴 - Прогресс опыта ребёнка до следующего ключика - Семейный банк: сколько баллов накоплено - Family streak - Кнопка «Новое задание» **Дальше:** навык → P-07 / уведомление → P-08 --- ### 🔍 P-07 — Детальный экран навыка **Что:** история выполнений, кривая прогресса, Ebbinghaus-повторение, рекомендация AI. --- ### 🔔 P-08 — Подтверждение выполнения **Что:** «[Имя] выполнил: [задание]». ✅ Подтвердить / ❓ Уточнить. Опциональный комментарий ребёнку. --- ### 🎁 P-09 — Уведомление о призе **Что:** «Мише выпала [приз] (🟡 редкий, 💰)». Кнопка «Реализовать» → инструкция или напоминание. **Почему:** родитель видит конкретное действие, которое нужно сделать. Не абстракция — конкретный шаг. --- ### 🛡️ P-10 — Управление отсрочками **Что:** список призов ребёнка. Для каждого — «Поставить на паузу» с длительностью и нарративом. Стоимость отображается до подтверждения. --- ### 📸 P-11 — Семейный ритуал (подтверждение) **Что:** семья реализовала приз → кнопка «Отметить» → загрузить фото → баллы начислены в семейный банк. **Почему:** фото = доказательство и воспоминание одновременно. Оно попадает в личную историю ребёнка и в семейный архив. --- ### 🎡 P-12 — Семейный барабан **Что:** анимация вращения. Весь экран — торжественный момент. Выпавший приз — крупно, с описанием. **Почему:** это семейный ритуал, а не технический экран. Дизайн должен это подчёркивать. --- ### 📋 P-13 — Weekly Report **Порядок имеет значение:** 1. Главное достижение ребёнка — одно, крупно 2. Прогресс по навыкам 3. Призы недели: что выпало, что реализовано 4. Семейный банк: прогресс 5. Рекомендация AI — одно действие 6. «Поделиться с [имя]» --- ## Screen-by-Screen: Ребёнок --- ### 🎮 C-01 — Приветственный экран **Что:** «Привет, [имя]. Ты уже назвал свои мечты. Выполняй квесты — зарабатывай ключики!» Анимация: барабан в ожидании, ключик на 0. **Дальше:** → C-02 --- ### 📖 C-02 — Туториал (3–4 экрана, пропускаемый) 1. Квест → выполни → «Готово» 2. Папа/мама подтверждают → XP твой 3. Накопи XP → уровень → ключик → барабан 4. Семья тоже играет. Вместе веселее. --- ### 🗺️ C-03 — Главный экран **Что:** - Прогресс XP до следующего уровня (вверху, всегда виден) - Ключики в кармане (сколько готово к барабану) - Family streak - 1–3 квеста на сегодня **Дальше:** квест → C-04 / ключик → C-06 --- ### 📋 C-04 — Экран квеста | Уровень ZPD | Что видит ребёнок | |---|---| | L3 — новичок | Пошаговая инструкция + иллюстрации | | L2 — знакомо | Чеклист без деталей | | L1 — умеет | Только цель | | L0 — мастер | Задание без инструкции | Кнопка «Готово» — активируется после минимального времени. --- ### 🎉 C-05 — Экран завершения квеста **Что:** - Анимация XP (+N) - Прогресс до следующего уровня - Строка Levera: «Три недели назад ты не умел этого. Теперь умеешь.» - Если уровень достигнут → «Ты вырос! Тебе выпал ключик 🗝️» --- ### 🎡 C-06 — Барабан (личный Wish Wheel) **Что:** - Анимация вращения ключиком - Торжественный момент: что выпало, уровень редкости (⚪/🟡/🔴), измерение (💰/⏱️/❤️) - «Папа/мама уже знают!» - Кнопка: «Напомнить» (если приз не реализован в течение N часов) Если приз на отсрочке → нарратив + автоматически следующий приз. --- ### ⏰ C-07 — Напоминание родителю **Что:** простой экран. «Папа, где моя [приз]? 😊» — отправляет мягкое уведомление. **Почему:** ребёнок учится просить, а не требовать. Тон — добрый, не обвинительный. --- ### 📸 C-08 — Семейный ритуал (ребёнок) **Что:** «Время семейного барабана! Зови всех.» Анимация ожидания. Затем — совместное вращение. После вращения → загрузить фото вместе → баллы начислены. --- ### 🌳 C-09 — Skill Tree Дерево навыков по категориям. Каждый узел — уровень + цвет. Ebbinghaus-повторения подсвечены. --- ### 📖 C-10 — История роста (личный дневник) **Что:** - Хронологическая лента: квесты, уровни, призы, фото - «Посмотри, каким ты был 3 месяца назад» - Визуализация роста навыков во времени **Почему:** это не геймификация. Это артефакт. Ребёнок, который видит собственный путь — понимает, что рост реален. Родитель, который листает эту ленту спустя год — не уходит из продукта никогда. --- ## Ключевые дизайн-решения | Решение | Обоснование | |---|---| | Три петли (личная → семейная → внешняя) | Каждая петля удерживает свой тип пользователя на своём временном горизонте | | Фото = подтверждение + воспоминание | Убивает двух зайцев: верификация реализации и создание семейного архива | | Напоминание от ребёнка, не от системы | Ребёнок учится коммуникации. Родитель слышит ребёнка, не робота. | | Кризисный механизм без стигмы | Семья с финансовыми трудностями не выпадает из продукта | | История роста как артефакт | Самый сильный retention-инструмент: данные, которые нельзя перенести | | Семейный барабан — торжественный ритуал | Не технический экран. Момент, ради которого собирается вся семья. | | Family streak — двустороннее подтверждение | Стрик не рвётся из-за забытого тапа | | Weekly Report начинается с достижения | Дизайн для гордости, не для отчётности | --- ## MVP-приоритет **Для Wizard-of-Oz теста — минимум:** 1. **C-03** — главный экран (quest-лист + прогресс XP) — вручную 2. **C-04** — экран квеста с уровнями ZPD 3. **C-05** — завершение квеста + анимация ключика при достижении уровня 4. **C-06** — барабан (прототип в Figma, результат — вручную) 5. **P-08** — подтверждение родителя (можно Telegram) 6. **P-09** — уведомление о призе (можно Telegram) 7. **P-13** — Weekly Report (можно PDF/Telegram) **После подтверждения W4 retention ≥25%:** - P-11/P-12 — семейный ритуал и семейный барабан - C-10 — история роста - P-04 — полная AI-сессия - Петля 3 — внешняя (партнёры, гранты)
