# Levera — W4 Retention Test Protocol *Протокол для теста с первыми 10–15 семьями. Цель: подтвердить или опровергнуть W4 retention ≥25% до масштабирования.* --- ## Что мы проверяем **Главная гипотеза:** семья, которая начала использовать Levera, возвращается на 4-й неделе. W4 retention = доля семей, активных на 4-й неделе после регистрации. **Порог:** - ✅ **≥25%** — механика работает, строить дальше - ⚠️ **15–24%** — есть сигнал, но нужна итерация (что именно не работает — см. диагностику) - ❌ **<15%** — механика не удерживает, нужен пересмотр до масштабирования --- ## Кого набирать **Профиль семьи:** - Ребёнок 7–12 лет - Родитель активно пользуется смартфоном - Родитель явно выражает интерес к развитию ребёнка (не просто «попробую») - Готов к еженедельному 10-минутному звонку **Как находить:** - Личные знакомые с детьми нужного возраста - Родительские чаты в Telegram/WhatsApp - Школьные/кружковые родительские группы **Красные флаги — не брать:** - Родитель скептичен с первого разговора - Ребёнок не в целевом возрасте (6 и младше, 13 и старше) - Семья явно перегружена — нет времени на новый инструмент **Размер выборки:** минимум 10 семей. Меньше — результаты статистически нерелевантны. --- ## Метрики ### Основная | Метрика | Определение | Цель | |---|---|---| | **W4 Retention** | % семей, у которых хотя бы 1 квест выполнен и подтверждён на 4-й неделе | ≥25% | ### Вспомогательные | Метрика | Что показывает | Хороший сигнал | |---|---|---| | **DAU/MAU ребёнка** | Насколько часто ребёнок открывает приложение | ≥40% | | **Quest completion rate** | % квестов, которые ребёнок завершает (не просто открывает) | ≥60% | | **Parent confirmation rate** | % квестов, подтверждённых родителем вручную (не авто) | ≥50% | | **Wheel spin rate** | % семей, которые прокрутили барабан хотя бы раз | ≥80% | | **Prize fulfillment rate** | % призов, которые родитель реализовал | ≥70% | | **Reminder sent** | % семей, где ребёнок нажал «Напомнить» | Информационная — не цель | | **Streak ≥7 days** | % семей со стриком 7+ дней к концу W4 | ≥30% | --- ## Расписание теста | Неделя | Что делает команда | Что фиксируем | |---|---|---| | **До старта** | Онбординг: 30-минутный звонок с каждой семьёй. Как ребёнок отреагировал?» **Для ребёнка (если есть возможность — спросить отдельно):** 1. «Тебе нравится крутить барабан? Что ты чувствуешь, когда крутишь?» 2. «Ты помнишь, что тебе выпало? Ты уже получил это?» 3. «Ты сам хочешь заходить в приложение или папа/мама напоминают?» --- ## Вопросы для W4-звонка (20 мин) Цель: retention, причины ухода/остаться, готовность платить. **Для родителя:** 1. «Вы всё ещё пользуетесь приложением? Как часто?» 2. «Что изменилось в поведении ребёнка за месяц? Есть ли что-то, что ты заметил?» 3. «Что тебе нравится больше всего?» 4. «Что тебе не нравится или чего не хватает?» 5. «Если бы приложение стоило 490 рублей в месяц — ты бы заплатил?» - Если «да»: «Сколько максимально?» - Если «нет»: «Что должно быть иначе, чтобы ты заплатил?» 6. «Ты бы порекомендовал это кому-то из друзей с детьми?» (NPS 0–10) --- ## Диагностика: что значит каждый результат ### Если W4 retention <25% Задать себе вопросы по воронке: ``` Quest completion rate <60%? → Проблема в квестах: слишком сложные, скучные или нерелевантные. Решение: пересмотреть ZPD-калибровку, добавить примеры квестов. Parent confirmation rate <50%? → Родитель выпадает из петли. Подтверждение — лишнее трение. Решение: снизить порог (авто-подтверждение по умолчанию, ручное — опция). Wheel spin rate <80%? → Дети не доходят до барабана. Уровень слишком долго набирается. Решение: снизить XP-порог для первого ключика. Prize fulfillment rate <70%? → Родитель не реализует призы. Либо слишком дорого, либо забывает. Решение: проверить список желаний — нет ли слишком дорогих. Добавить ❤️/⏱️ призы. Всё выше норм, но retention всё равно низкий? → Проблема не в механике, а в привычке. Нет достаточного триггера для ежедневного входа. Решение: пересмотреть push-стратегию и триумф-момент. ``` ### Если W4 retention ≥25% Проверить дополнительно перед масштабированием: - **NPS ≥ 7** у большинства родителей → можно рекомендовать - **Готовность платить ≥490 руб.** у ≥50% → юнит-экономика работает - **Хотя бы 1 семья сама привела другую семью** → органический рост возможен --- ## Что НЕ делать во время теста | Не делать | Почему | |---|---| | Помогать семьям «слишком сильно» | Исказит результат — нужно видеть, что происходит без поддержки основателя | | Вносить изменения в продукт в процессе теста | Нельзя сравнивать результаты разных семей при разном продукте | | Набирать только лояльных знакомых | Смещение выборки. Минимум 30% — незнакомые или «тёплые», но не близкие | | Считать retention только по открытиям приложения | Открытие ≠ использование. Считать только выполненные + подтверждённые квесты | --- ## Таблица для фиксации результатов | Семья | W1 квесты | W2 квесты | W3 квесты | W4 квесты | Барабан | Приз реализован | NPS | Платить? | |---|---|---|---|---|---|---|---|---| | Семья 1 | | | | | | | | | | Семья 2 | | | | | | | | | | ... | | | | | | | | | **Считать W4 retention:** кол-во семей с ≥1 квестом на W4 / общее кол-во семей × 100%
