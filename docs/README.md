# Levera documentation

All project documentation is grouped here so the repository root remains available for application code and infrastructure.

## Sections

- `product/` — canonical product definition, roadmap, UX and onboarding.
- `research/` — competitors, market evidence, audience and raw exports.
- `validation/` — interview and experiment protocols.
- `technical/` — technical specifications.
- `brand/` — brand identity.
- `marketing/` — acquisition, content, store and PR materials.
- `business/` — business model, pricing, partnerships and fundraising.
- `legal/` — legal drafts requiring professional review.
- `archive/` — superseded or incomplete materials.

## Canonical documents

- Live execution plan: `EXECUTION-PLAN.md`
- Product source of truth: `product/Product-Bible.html`
- Active personal-first direction: `product/PERSONAL-FIRST-DIRECTION.md`
- Research entry point: `research/README.md`
- MVP implementation brief: `technical/MVP-Technical-Brief.html`
