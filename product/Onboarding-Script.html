# Levera — Onboarding Script *Версия 1.0 — Точные тексты и сценарии первых 5 минут* --- > **Как использовать этот документ** > > Каждый экран онбординга описан тремя слоями: (1) что видит пользователь, (2) точный текст на экране, (3) почему именно так. Третий слой — не для UI, он для команды. Чтобы никто не менял формулировки без понимания причины. > > Правило одно: онбординг должен занимать ≤5 минут. Если это нарушено — что-то лишнее. --- ## Архитектура онбординга ``` Родитель скачивает приложение │ ▼ [Блок A — Вход и установка доверия] A-1. Экран приветствия A-2. Регистрация │ ▼ [Блок B — Ребёнок] B-1. Добавление ребёнка B-2. Совместный ритуал: список желаний ребёнка B-3. Родитель одобряет желания B-4. Родитель составляет свой список │ ▼ [Блок C — Первая цель] C-1. AI-сессия: что хочешь развить? C-2. Первые квесты — одобрение │ ▼ [Блок D — Передача ребёнку] D-1. Экран передачи устройства D-2. Приветствие ребёнка D-3. Краткий туториал (3 шага) D-4. Первый квест │ ▼ Готово. Первый квест активен. ``` **Цель онбординга:** довести до первого выполненного квеста. Не до регистрации. Не до подписки. До первого реального действия. --- ## Блок A — Вход и установка доверия --- ### A-1. Экран приветствия **Что видит пользователь:** Минималистичный экран. Логотип. Одна строка. Две кнопки. --- **Текст на экране:** ``` LEVERA Твой путь. Твой темп. Твой результат. [Начать] [Уже есть аккаунт] ``` --- **Почему так:** Первый экран задаёт тон. Positioning-строка — три слова с "твой" — сразу ставит ownership на пользователя, а не на приложение. Нет обещаний. Нет списка фич. Нет "Добро пожаловать в Levera — лучшее приложение для развития детей!". Тишина говорит больше. --- ### A-2. Регистрация **Что видит пользователь:** Минимальная форма. Телефон или email. Код подтверждения. Ничего лишнего. --- **Текст на экране:** ``` Введи номер телефона [+7 ___ ___ __ __] Мы отправим код. Это займёт секунду. [Продолжить] ``` *(После ввода кода)* ``` [_ _ _ _ _ _] Код отправлен на +7 XXX XXX XX XX [Изменить номер] ``` --- **Почему так:** Никаких имён, паролей, дат рождения на этом шаге. Один барьер — телефон. Любое дополнительное поле на этом экране снижает конверсию. Имя и профиль — позже, когда пользователь уже внутри. --- ## Блок B — Ребёнок --- ### B-1. Добавление ребёнка **Что видит пользователь:** Простая форма. Имя, возраст. Опционально — фото. --- **Текст на экране:** ``` Как зовут твоего ребёнка? [Имя ребёнка] Сколько ему / ей лет? [ 6 ] ← → [ 17 ] (слайдер) [Добавить фото] ← необязательно [Продолжить] ``` --- **Подсказка под слайдером (появляется при выборе возраста):** | Возраст | Подсказка | |---|---| | 6–8 | "Первые навыки. Простые задания. Много поддержки." | | 9–11 | "Самый быстрый период роста. Хорошее время начать." | | 12–14 | "Важен контроль сложности — не слишком легко, не слишком сложно." | | 15–17 | "Больше автономии. Меньше инструкций. Больше ответственности." | --- **Почему так:** Подсказка по возрасту делает две вещи одновременно: объясняет логику продукта и создаёт ощущение, что приложение "знает" этот возраст. Это доверие до первого квеста. --- ### B-2. Совместный ритуал: список желаний ребёнка Это первый момент в онбординге, где родитель должен позвать ребёнка. Это ключевой переход. **Что видит пользователь:** Экран-инструкция перед ритуалом. --- **Текст на экране:** ``` Теперь — вместе Позови [имя]. Пусть скажет всё, чего хочет. Без ограничений. Без фильтров. Это его список. Ты посмотришь потом. [Готово, [имя] рядом] ``` --- **Почему так:** "Без ограничений. Без фильтров." — это важная инструкция. Родители склонны сразу редактировать. Мы говорим: сначала ребёнок говорит всё — потом ты решаешь. Это меняет динамику: ребёнок чувствует, что его слышат. --- **Следующий экран — ввод желаний ребёнком:** ``` [Имя], что ты хочешь? Говори или пиши — всё что угодно. [🎤 Говорить] [✏️ Написать] Уже добавлено: ┌─────────────────────────┐ │ 🎮 Новый геймпад │ │ 🍕 Пицца с семьёй │ │ 🐕 Взять собаку │ └─────────────────────────┘ [Готово, это всё] ``` --- **Детали интерфейса:** - Каждое желание появляется карточкой сразу после ввода — позитивная обратная связь - Кнопка "Ещё" подталкивает добавить больше: "Добавь ещё одно — чем больше, тем интереснее барабан" - Минимум: 3 желания. Если меньше — мягкое напоминание, не блокировка --- ### B-3. Родитель одобряет желания **Переход обратно к родителю:** ``` Хорошо. Теперь твоя очередь. Посмотри, что [имя] хочет. Для каждого — реши: да или позже. И выбери, насколько это редко. [Смотреть список] ``` --- **Экран одобрения (одна карточка за раз):** ``` ┌─────────────────────────────────┐ │ │ │ 🐕 Взять собаку │ │ │ │ Насколько это редкий приз? │ │ │ │ ⚪ Обычный ~70% шансов │ │ 🟡 Редкий ~25% шансов │ │ 🔴 Очень редкий ~5% шансов │ │ │ │ Что это стоит? │ │ 💰 Деньги ⏱️ Время ❤️ Эмоции │ │ │ │ [✅ Добавить] [❌ Убрать] │ │ │ └─────────────────────────────────┘ Желание 3 из 5 ``` --- **Подсказки под уровнями редкости (появляются при тапе):** - ⚪ Обычный — "Выпадает часто. Шоколадка, наклейки, любимое блюдо." - 🟡 Редкий — "Выпадает иногда. Кино, пиццерия, новая книга." - 🔴 Очень редкий — "Выпадает редко, но гарантированно раз в [период]. Может быть дорогим по деньгам — или просто особенным." **Подсказки под измерениями ценности:** - 💰 Деньги — "Ты тратишь деньги." - ⏱️ Время — "Ты тратишь время вместе с ним." - ❤️ Эмоции — "Ты отдаёшь внимание — обнимашки, особый ужин, семейный ритуал." --- **Почему так:** Одна карточка за раз — не список. Родитель принимает решение по каждому желанию отдельно, осознанно. Ребёнок видит потом: папа / мама прочитали каждое. Это и есть первый момент доверия. --- ### B-4. Родитель составляет свой список **Переходный экран:** ``` И ещё кое-что. [Имя] тоже может делать тебе приятное. Что ты хочешь от него? Это не задания. Это — его подарки тебе. [Составить свой список] [Позже] ``` --- **Экран ввода (такой же, как у ребёнка):** ``` Что тебе важно? Подсказки: ┌──────────────────────────────────┐ │ ⏱️ Тихий час — [имя] играет сам │ │ ❤️ Завтрак в постель │ │ ❤️ Обнялись без повода │ │ 💰 Убрал сам, без напоминания │ └──────────────────────────────────┘ [🎤 Говорить] [✏️ Написать] [Готово] ``` --- **Почему так:** Родительский список — это не обязательный шаг. Кнопка "Позже" всегда видна. Но если родитель составит список — симметрия заложена с первого дня. Ребёнок увидит: у папы / мамы тоже есть мечты. Это меняет всё. --- ## Блок C — Первая цель --- ### C-1. AI-сессия: что хочешь развить? **Переходный экран:** ``` Отлично. Теперь скажи мне — что хочешь развить в [имени]? Говори как есть. Одно или несколько. Я разберусь. [🎤 Говорить] [✏️ Написать] ``` --- **Тапабельные подсказки (видны под полем ввода):** ``` Не знаешь с чего начать? Попробуй: → "Хочу, чтобы сам убирал за собой" → "Боится ответственности" → "Не умеет планировать время" → "Хочу научить готовить базовые вещи" → "Проблемы с учёбой — откладывает всё на потом" ``` --- **Ответ AI (после ввода — если цель конкретная):** *Пример входа: "Хочу, чтобы сам убирал комнату"* ``` Понял. Для [имени] (10 лет) это — новый навык. Начнём с малого — простые задания с инструкцией. Постепенно усложним, когда он освоится. Вот первые квесты: ┌─────────────────────────────────────┐ │ 🏠 Убери стол перед сном │ │ Уровень: начинающий │ │ Время: ~5 мин │ │ │ │ [Посмотреть инструкцию] │ └─────────────────────────────────────┘ ┌─────────────────────────────────────┐ │ 🏠 Сложи одежду после стирки │ │ Уровень: начинающий │ │ Время: ~10 мин │ │ │ │ [Посмотреть инструкцию] │ └─────────────────────────────────────┘ [Принять все] [Изменить] ``` --- **Ответ AI (если цель слишком широкая — задаёт один вопрос):** *Пример входа: "Хочу, чтобы стал ответственным"* ``` Ответственность — это большая тема. Что важнее всего прямо сейчас: дом и быт, учёба, или обещания и договорённости? [Дом и быт] [Учёба] [Обещания] [Всё вместе] ``` --- **Почему так:** AI задаёт максимум один вопрос. Не анкету. Не опросник из 10 пунктов. Один вопрос — и сразу квесты. Это ощущение скорости и интеллекта. --- ### C-2. Одобрение первых квестов **Экран одобрения:** ``` Первые квесты [имени] готовы. Убери лишнее. Добавь своё. Или просто нажми — запустить. [список квестов] [🚀 Запустить] ``` **Редактирование квеста (тап на карточку):** ``` ┌─────────────────────────────────────┐ │ Убери стол перед сном │ │ │ │ Описание: │ │ [Убери всё с поверхности стола. │ │ Протри влажной салфеткой. │ │ Положи вещи на место.] │ │ │ │ Уровень: [Начинающий ▼] │ │ Время: [5 мин ▼] │ │ │ │ [Сохранить] [Удалить квест] │ └─────────────────────────────────────┘ ``` --- ## Блок D — Передача ребёнку --- ### D-1. Экран передачи устройства Это переломный момент онбординга. Родитель должен передать телефон ребёнку. **Текст на экране:** ``` Готово. Теперь передай телефон [имени]. Пусть он сам зайдёт первый раз. Это важно — его первый шаг должен быть его. [Передать [имени]] ``` --- **Почему так:** "Его первый шаг должен быть его" — это не красивая фраза. Это педагогический принцип. Ребёнок, которого запустили в приложение без него, чувствует: это родительская идея, не моя. Ребёнок, который зашёл сам — уже принял решение. --- ### D-2. Приветствие ребёнка Ребёнок берёт телефон. Видит другой экран — сразу. **Текст на экране:** ``` Привет, [имя]. Ты уже надиктовал свои мечты. Первый квест — и первая крутилка твоя. [Смотреть мои квесты] ``` --- **Почему так:** Три предложения. Первое — узнавание. Второе — напоминание о том, что он уже сделал (и что это важно). Третье — действие. Никакого "Добро пожаловать в Levera!" Никакого туториала сразу. Сначала — эмоция. --- ### D-3. Краткий туториал (3 экрана, пропускаемый) Туториал показывается **после** первого знакомства — не до. Пропускается одним свайпом. --- **Экран 1:** ``` Как это работает Выполни квест → Нажми "Готово" → Папа / мама подтвердят → Крутилка твоя. ●○○ [Дальше] [Пропустить] ``` --- **Экран 2:** ``` Накопи крутилки Крутилки копятся. Когда накопишь достаточно — крутишь барабан. Барабан выдаёт призы из твоего списка. Не всегда те, что ожидаешь. Иногда лучше. ○●○ [Дальше] [Пропустить] ``` --- **Экран 3:** ``` Папа / мама тоже играют У них тоже есть барабан. Ты можешь дать им опыт — если заметишь что-то хорошее. Это не задание. Это твой выбор. ○○● [Понял. Начинаем.] ``` --- **Почему так:** Три экрана — ровно три механики, которые ребёнку нужно знать. Квест → крутилка → барабан. Симметрия с родительским барабаном объясняется с первого дня — не как сюрприз позже. "Это твой выбор" — уважение к автономии ребёнка заложено уже здесь. --- ### D-4. Первый квест **Главный экран ребёнка (после туториала):** ``` Привет, [имя] 👋 Сегодня ┌─────────────────────────────────────┐ │ 🏠 Убери стол перед сном │ │ │ │ ○ ─────────────────── 0 / 1 │ │ │ │ [Открыть квест] │ └─────────────────────────────────────┘ Крутилки: 0 / 5 до барабана Family Streak: 0 дней ``` --- **Экран самого квеста (L3 — новичок):** ``` Убери стол перед сном Вот что нужно сделать: ☐ 1. Убери всё с поверхности стола ☐ 2. Протри стол влажной салфеткой ☐ 3. Положи вещи на свои места ☐ 4. Проверь: на столе ничего лишнего? Примерно 5 минут. [✅ Готово] ``` --- **Экран после нажатия "Готово":** ``` Отлично. Ждём, пока папа / мама подтвердят. Обычно это занимает несколько минут. Крутилка будет твоей, как только они нажмут. [← Назад к квестам] ``` --- ## Уведомления онбординга ### Родителю — сразу после завершения онбординга: ``` Push: "Levera запущена. [Имя] видит свой первый квест. Как только он нажмёт «Готово» — тебе придёт запрос на подтверждение." ``` ### Родителю — когда ребёнок нажал "Готово": ``` Push: "[Имя] говорит, что сделал: «Убери стол перед сном». Подтверди 👍" ``` ### Ребёнку — когда родитель подтвердил: ``` Push: "Засчитано. +1 крутилка 🎯 Ещё 4 до барабана." ``` --- ## Экраны возврата (после паузы) Эти экраны — не часть первого онбординга. Но часть того же скрипта: что говорит Levera, когда пользователь возвращается после перерыва. --- ### Ребёнок не заходил 3+ дня: ``` С возвращением, [имя]. [Квест дня] [Открыть] ``` Без драмы. Без "ты пропустил 3 дня!". Просто — квест. --- ### Родитель не заходил 5+ дней: ``` [Имя] ждёт подтверждения с [дата]. [Посмотреть] ``` Один факт. Одно действие. Никакого обвинения. --- ### Семья не заходила 7+ дней: ``` Тихая неделя. Если что-то случилось — всё в порядке. Квесты ждут, когда будете готовы. [Продолжить] ``` --- ## Полный список текстов: карманная версия Для разработчиков и копирайтеров — все key strings в одном месте. ### Системные строки | Ключ | Текст | |---|---| | `onboarding.welcome.headline` | Твой путь. Твой темп. Твой результат. | | `onboarding.welcome.cta` | Начать | | `onboarding.phone.prompt` | Введи номер телефона | | `onboarding.phone.hint` | Мы отправим код. Это займёт секунду. | | `onboarding.child.name` | Как зовут твоего ребёнка? | | `onboarding.child.age` | Сколько ему / ей лет? | | `onboarding.wishlist.intro` | Позови [имя]. Пусть скажет всё, чего хочет. | | `onboarding.wishlist.prompt` | [Имя], что ты хочешь? | | `onboarding.parent.transfer` | Передай телефон [имени]. | | `onboarding.child.welcome` | Привет, [имя]. Ты уже надиктовал свои мечты. | | `quest.complete.pending` | Ждём, пока папа / мама подтвердят. | | `quest.confirmed` | Засчитано. +1 крутилка 🎯 | | `return.child.brief` | С возвращением, [имя]. | | `return.parent.pending` | [Имя] ждёт подтверждения с [дата]. | | `return.family.long` | Тихая неделя. Квесты ждут, когда будете готовы. | --- ### Push-уведомления | Ключ | Текст | Получатель | |---|---|---| | `push.quest_done` | [Имя] говорит, что сделал: «[квест]». Подтверди 👍 | Родитель | | `push.quest_confirmed` | Засчитано. +1 крутилка 🎯 Ещё [N] до барабана. | Ребёнок | | `push.level_up` | [Имя] вырос до уровня [N]. Ключик получен 🗝️ | Оба | | `push.prize_won` | [Имя] выиграл: [приз]. Реализуй в ближайшее время 🎁 | Родитель | | `push.weekly_report` | Отчёт недели [имени] готов. Посмотри, как он вырос. | Родитель | | `push.reminder_gentle` | [Имя] спрашивает про [приз] 😊 | Родитель | --- ### Triumph-строки (банк) Используются после завершения квеста. Выбираются по контексту. **Базовые (каждый квест):** ``` "Сделано. +[N] XP." "Ещё один." "Не пропустил." "[N] дней подряд." ``` **Внутренние milestone (формула роста):** ``` "Три недели назад ты не умел этого. Теперь умеешь." "Вчера ты тоже мог пропустить. Но не пропустил." "Это стало твоим. Навсегда." "[N] раз подряд. Это уже не случайность." ``` **Внешние milestone (только при росте уровня, streak кратном 7, первом выполнении навыка):** ``` "97% детей не делают этого каждый день. Ты делаешь." "Не все дети доходят до уровня [N]. Ты дошёл." "Сегодня ты сделал то, что многие взрослые откладывают." "Это не случайность. Это ты." ``` **Правило выбора:** - Базовые — каждый квест (ротация случайная) - Внутренние — при streak 3+, после первых 5 выполнений навыка - Внешние — только milestone: новый уровень / streak ×7 / первое выполнение навыка --- ### Строки онбординга: ошибки и крайние случаи | Ситуация | Текст | |---|---| | Ребёнок добавил менее 3 желаний | "Добавь ещё одно — чем больше желаний, тем интереснее барабан." | | Родитель убрал все желания ребёнка | "Оставь хотя бы одно — иначе барабану нечего выдавать." | | AI не понял запрос | "Попробуй сформулировать иначе — или выбери один из примеров ниже." | | Ребёнок пропустил туториал | Показать подсказку на первом квесте: "Первый раз?
